## What This Project Covers

Most recommendation tutorials implement just one technique. This project implements **three**, side by side, so you can see exactly how each approach differs in logic, output, and use case:

| Technique | How it works | Real-world example |
|---|---|---|
| **Demographic filtering** | Ranks items by popularity + rating score for all users | Netflix "Trending Now", "Top 10 in India" |
| **Content-based filtering** | Recommends similar items based on metadata (plot, genre, cast) | "Because you watched Inception..." |
| **Collaborative filtering** | Recommends based on what similar users liked | Spotify "Made for You" playlists |

## Notebooks

### `movie-recommendation.ipynb`
Full movie recommendation pipeline using **The Movies Dataset** (TMDB / MovieLens).

Covers all three filtering techniques:

**1. Demographic filtering**
- Uses IMDB's Weighted Rating (WR) formula to rank movies fairly across different vote counts
- Builds both a general top-movies chart and genre-specific charts
- Formula used:

```
WR = (v / (v + m)) × R + (m / (v + m)) × C

Where:
  v = number of votes for the movie
  m = minimum votes required to be listed (95th percentile)
  R = average rating of the movie
  C = mean rating across all movies
```

This prevents a movie with a 9.5 rating from 4 votes outranking a movie with an 8.2 rating from 10,000 votes.

**2. Content-based filtering (overview-based)**
- TF-IDF vectorisation on movie plot descriptions (overview field)
- Cosine similarity matrix computed across all movies
- Given a movie title, returns the top N most similar movies by plot

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import linear_kernel

tfidf = TfidfVectorizer(stop_words='english')
tfidf_matrix = tfidf.fit_transform(df['overview'])
cosine_sim = linear_kernel(tfidf_matrix, tfidf_matrix)
```

**3. Content-based filtering (metadata soup)**
- Builds a "soup" combining: cast, director, genres, keywords
- CountVectorizer + cosine similarity on the combined metadata string
- Penalises overrepresented cast members (stops popular actors from dominating results)

**4. Collaborative filtering**
- Uses the `Surprise` library with SVD (Singular Value Decomposition)
- Trained on `ratings_small.csv` from MovieLens
- Predicts personalised ratings for movies a user hasn't seen yet
- Outputs top N personalised recommendations per user ID

### `anime-recommendation.ipynb`
Recommendation engine applied to **anime data** — demonstrating the same techniques generalise beyond movies to any item catalogue.

## Key Concepts Demonstrated

- **IMDB Weighted Rating formula** — fairer ranking than raw averages
- **TF-IDF (Term Frequency-Inverse Document Frequency)** — weights words by how distinctive they are to a document vs. the whole corpus
- **Cosine similarity** — measures the angle between two vectors to find how similar two items are, independent of their magnitude
- **Metadata soup** — combining multiple features (cast, director, genre, keywords) into a single text field for unified vectorisation
- **SVD (Singular Value Decomposition)** — matrix factorisation technique that decomposes the user-item ratings matrix to predict unseen ratings
- **Cold-start problem** — why demographic filtering is useful when you have no user history

## Why This Project Matters for Any Business / Distribution Analytics

Recommendation logic isn't just for Netflix. The same principles apply directly to business analytics:

- **Demographic filtering** → "Top-selling products in this territory this quarter" (the same weighted-ranking logic)
- **Content-based filtering** → "Suggest similar products to what a distributor typically orders" (based on product metadata)
- **Collaborative filtering** → "Distributors with similar order patterns also buy these SKUs" (cross-sell opportunity detection)

This project demonstrates the ML foundation that can be applied to product recommendation engines in pharma distribution.
